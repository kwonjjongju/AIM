## 💡 아이디어 메모

### 핵심 컨셉
> "화이트보드 + 포스트잇을 온라인으로 옮긴 느낌"

- 경영자가 회의실 화이트보드 앞에서 포스트잇 훑어보는 경험을 디지털로
- 복잡한 분석 도구 ❌ → 단순한 "구경하는 지도" ✅
- 숫자는 "정보"지 "평가"가 아님

---

## 🎯 경영자 관점 핵심 질문

경영자가 이 시스템을 통해 답을 얻고 싶은 질문들:

1. **"요즘 어디 부서에서 어떤 얘기들이 많이 올라오지?"**
   - → 부서별 현황 요약 필요

2. **"이 중에서 아직 손도 못 댄 건 뭐야?"**
   - → 💡 떠올림 상태 필터 필요

3. **"오래 걸리는 건 왜 오래 걸릴까?"**
   - → 오래된 항목 하이라이트 필요

4. **"이건 굳이 지금 안 해도 되는 거 아냐?"**
   - → ⏸️ 멈춤 상태 리스트 필요

5. **"이미 끝난 건 어떤 식으로 정리됐지?"**
   - → ✅ 정리됨 상태 조회 필요

## 🚫 절대 하지 말아야 할 것

### Anti-Patterns

```
❌ KPI, 점수, 평가, 실적 비교
❌ 부서 간 성과 랭킹
❌ 개인 평가 연동
❌ 퍼센트, 달성률
❌ 딱딱한 용어 (분석중, 검증, 성과확인)
❌ 복잡한 승인 프로세스
```

### 피해야 할 단어
| ❌ 피할 단어 | ✅ 대안 |
|-------------|---------|
| 관리 | 보드, 리스트 |
| 성과 | 정리됨 |
| 평가 | (사용 안함) |
| 분석 | 훑어보기 |
| 등록됨 | 떠올림 |
| 진행중 | 만지는 중 |
| 완료 | 정리됨 |

## 🎨 톤앤매너 아이디어

### 등록 화면 문구
- 제목: **"뭐가 불편한가요?"**
- 메모: **"편하게 써주세요 (형식 없음)"**
- 버튼: **[일단 올리기]**

### 철학
> "나중에 다듬자"가 기본

- 최소 필수 입력만
- 빈칸 OK
- "왜 안 적었냐" 질문 안 나오게

## 📊 대시보드 아이디어

### 상단 요약
```
전체 ○건 / 정리됨 ○건 / 멈춤 ○건
```

### 우측 사이드
- 부서별 개수 작은 바
- 오래된 항목 TOP 5

### 중앙
- 카드 리스트 (최신순)
- 포스트잇 느낌 카드

|------|--------|------|------|

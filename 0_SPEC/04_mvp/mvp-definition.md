# MVP 정의 (MVP Definition)
## 업무 개선 보드 시스템

---

## 1. MVP 개요

### 1.1 MVP 목표
> **"4주 내에 현장에서 사용 가능한 업무 개선 리스트 시스템 출시"**

### 1.2 핵심 가치
| 가치 | 설명 |
|------|------|
| 현장 가시화 | 회사 전체 개선 활동을 한눈에 파악 |
| 자유로운 등록 | 부담 없이 개선 아이디어 제안 |
| 캐주얼한 경험 | 성과 평가 아닌 리스트 관리 느낌 |

## 3. MVP 사용자 스토리

### 3.1 일반 직원
```gherkin
AS A 일반 직원
I WANT TO 불편한 점을 쉽게 등록하고 싶다
SO THAT 내 의견이 회사에 전달될 수 있다

Acceptance Criteria:
- 로그인 후 [일단 올리기] 버튼 클릭
- 제목만 입력하면 등록 가능
- 내 항목은 수정/삭제 가능
- 상태 변경 가능 (💡→👀→🛠️→⏸️→✅)
```

### 3.2 부서 담당자
```gherkin
AS A 부서 담당자
I WANT TO 우리 부서 항목을 관리하고 싶다
SO THAT 부서 내 개선 활동을 조율할 수 있다

Acceptance Criteria:
- 부서 탭에서 우리 부서 항목만 조회
- 부서 내 항목 상태 변경 가능
- 부서 내 항목 수정/삭제 가능
```

### 3.3 경영자
```gherkin
AS A 경영자
I WANT TO 전사 개선 현황을 한눈에 보고 싶다
SO THAT 현장의 온도를 느낄 수 있다

Acceptance Criteria:
- 대시보드에서 전체 현황 확인
- 부서별 건수 확인
- 오래된 항목 확인
- 상태별 건수 확인
```

## 4. MVP 화면 목록

|------|------|----------|

## 5. MVP 기술 범위

### 5.1 포함
- React SPA + TypeScript
- Node.js + Express API
- PostgreSQL 데이터베이스
- Redis 세션 저장소
- JWT 인증
- Docker 배포

### 5.2 제외
- CI/CD 자동화 (수동 배포)
- 모니터링 시스템 (기본 로그만)
- 성능 최적화 (기본 수준)

## 6. MVP 일정

```
Week 1: 기반 구축
├── Day 1-2: 프로젝트 설정, DB 스키마
├── Day 3-4: 인증 API, 기본 UI
└── Day 5: 항목 CRUD API

Week 2: 핵심 기능
├── Day 1-2: 항목 등록/조회 UI
├── Day 3-4: 상태 관리, 이력
└── Day 5: 부서 탭, 필터링

Week 3: 대시보드 & 마무리
├── Day 1-2: 대시보드 구현
├── Day 3-4: 권한 관리, 반응형
└── Day 5: 테스트, 버그 수정

Week 4: 배포 & 안정화
├── Day 1-2: Docker 배포
├── Day 3-4: UAT, 피드백 반영
└── Day 5: 운영 준비, 출시
```

## 7. MVP 리스크

| 리스크 | 영향 | 대응 |
|--------|------|------|
| 인증 구현 지연 | 전체 일정 지연 | 초기 집중, 간소화 |
| 요구사항 추가 요청 | 범위 확대 | MVP 외 항목은 Phase 2로 |
| 반응형 UI 이슈 | 모바일 사용 불가 | 데스크톱 우선, 모바일 기본 |

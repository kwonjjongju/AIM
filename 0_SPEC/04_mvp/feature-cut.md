## 3. Phase별 기능 요약

### Phase 1: MVP (2주)

```
✅ 인증 (로그인/로그아웃/토큰)
✅ 항목 CRUD (제목, 메모)
✅ 상태 관리 (5단계, 이력)
✅ 부서별 탭, 상태별 필터
✅ 기본 대시보드
✅ 반응형 UI
```

### Phase 2: 핵심 고도화 (1주)

```
✅ 담당자 지정
✅ 관련 부서 태그
✅ 파일 첨부
✅ 상태 변경 메모
✅ 기간별/오래된 항목 필터
✅ 부서/사용자 관리 UI
```

### Phase 3: 선택 기능 (1주+)

```
✅ SSO 연동
✅ 알림 기능
✅ 검색
✅ 버블 맵, 타임라인
✅ 고급 시각화
```

## 4. 기능 컷 의사결정

### 4.1 MVP 컷 기준
| 기준 | 설명 |
|------|------|
| 핵심 가치 | 개선 항목 등록/조회/상태 관리 |
| 최소 사용성 | 로그인, 기본 UI |
| 구현 복잡도 | 낮음 ~ 중간 |

### 4.2 P2로 이연 사유
| 기능 | 이연 사유 |
|------|-----------|
| 파일 첨부 | 스토리지 설정, 복잡도 |
| 담당자 지정 | 핵심 가치 아님 |
| 관리자 UI | DB 직접 관리 가능 |

### 4.3 P3로 이연 사유
| 기능 | 이연 사유 |
|------|-----------|
| SSO 연동 | 외부 시스템 의존 |
| 알림 | 인프라 추가 필요 |
| 고급 시각화 | 시간 소요 |

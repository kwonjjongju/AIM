# ID 체계 규칙
## 업무 개선 보드 시스템

---

## 1. 개요

본 문서는 프로젝트 전반에서 사용되는 ID 체계를 정의한다.

---

## 2. ID 접두사 정의

### 2.1 요구사항 ID (REQ)

| 접두사 | 용도 | 예시 |
|--------|------|------|
| REQ-FR-### | 기능 요구사항 | REQ-FR-001 |
| REQ-NFR-### | 비기능 요구사항 | REQ-NFR-001 |
| REQ-SEC-### | 보안 요구사항 | REQ-SEC-001 |
| REQ-UI-### | UI 요구사항 | REQ-UI-001 |

### 2.2 설계 ID (DSN)

| 접두사 | 용도 | 예시 |
|--------|------|------|
| DSN-### | 설계 요소 | DSN-001 |
| DSN-###-API | API 설계 | DSN-001-API |
| DSN-###-DB | DB 설계 | DSN-001-DB |
| DSN-###-SEC | 보안 설계 | DSN-001-SEC |

### 2.3 작업 ID (TASK)

| 접두사 | 용도 | 예시 |
|--------|------|------|
| TASK-### | 개발 작업 | TASK-001 |
| TASK-###-FE | 프론트엔드 작업 | TASK-001-FE |
| TASK-###-BE | 백엔드 작업 | TASK-001-BE |

### 2.4 테스트 ID (TC)

| 접두사 | 용도 | 예시 |
|--------|------|------|
| TC-### | 테스트 케이스 | TC-001 |
| TC-###-UNIT | 단위 테스트 | TC-001-UNIT |
| TC-###-INT | 통합 테스트 | TC-001-INT |
| TC-###-E2E | E2E 테스트 | TC-001-E2E |

### 2.5 결정 ID (D)

| 접두사 | 용도 | 예시 |
|--------|------|------|
| D-### | 설계 결정 | D-001 |

## 3. ID 발급 규칙

### 3.1 번호 체계
- 3자리 숫자 사용 (001 ~ 999)
- 순차적으로 발급
- 삭제된 ID는 재사용하지 않음

### 3.2 하위 ID
- 상위 ID에 하이픈(-) 후 세부 번호 추가
- 예: REQ-FR-001-01, REQ-FR-001-02

### 3.3 카테고리 확장
- 필요 시 카테고리 추가 가능
- 기존 체계와 충돌하지 않도록 주의

---

## 4. 추적성 체인

```
REQ-### → DSN-### → TASK-### → TC-###
   ↓          ↓          ↓         ↓
요구사항    설계      작업     테스트
```

### 4.1 예시

| 요구사항 | 설계 | 작업 | 테스트 |
|----------|------|------|--------|
| REQ-FR-001 | DSN-001 | TASK-001 | TC-001 |
| 항목 등록 | 등록 API | 등록 구현 | 등록 테스트 |

## 5. ID 현황

### 5.1 요구사항 ID 현황

| 범위 | 용도 | 사용 현황 |
|------|------|-----------|
| REQ-FR-001 ~ 007 | 기능 요구사항 | 7개 사용 |
| REQ-NFR-001 ~ 003 | 비기능 요구사항 | 3개 사용 |
| REQ-SEC-001 | 보안 요구사항 | 1개 사용 |
| REQ-UI-001 ~ 006 | UI 요구사항 | 6개 사용 |

### 5.2 설계 ID 현황

| 범위 | 용도 | 사용 현황 |
|------|------|-----------|
| DSN-001 ~ 010 | 설계 요소 | 예약 |

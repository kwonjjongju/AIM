# 용어 정의 (Glossary)
## 업무 개선 보드 시스템

---

## 1. 비즈니스 용어

| 용어 | 영문 | 정의 |
|------|------|------|
| 개선 항목 | Improvement Item | 직원이 등록하는 업무 개선 아이디어/불편사항/과제 |
| 카드 | Card | 개선 항목을 포스트잇처럼 시각화한 UI 요소 |
| 보드 | Board | 카드들이 모여있는 화면 (화이트보드 느낌) |
| 상태 | Status | 개선 항목의 진행 단계 (💡👀🛠️⏸️✅) |
| 훑어보기 | Overview | 분석이 아닌 가볍게 현황을 파악하는 행위 |
| 정리됨 | Done | 개선 항목이 완료된 상태 (성과 대신 사용) |
| 떠올림 | Idea | 아이디어 단계의 개선 항목 |

---

## 2. 상태 용어

| 상태 | 아이콘 | 영문 코드 | 설명 |
|------|--------|-----------|------|
| 떠올림 | 💡 | IDEA | 아이디어 단계, 아직 시작 안함 |
| 보고 있음 | 👀 | REVIEWING | 누군가 검토 중 |
| 만지는 중 | 🛠️ | IN_PROGRESS | 실제 작업 진행 중 |
| 잠깐 멈춤 | ⏸️ | ON_HOLD | 일시 보류, 나중에 다시 |
| 정리됨 | ✅ | DONE | 완료, 해결됨 |

---

## 3. 사용자 역할 용어

| 역할 | 영문 | 설명 |
|------|------|------|
| 일반 직원 | EMPLOYEE | 개선 항목을 등록하고 본인 항목을 관리하는 사용자 |
| 부서 담당자 | DEPT_MANAGER | 부서 내 항목을 관리하는 사용자 |
| 경영자 | EXECUTIVE | 전체 현황을 조회하는 사용자 (등록/수정 권한 없음) |
| 시스템 관리자 | ADMIN | 시스템 전체를 관리하는 사용자 |

---

## 4. 기술 용어

| 용어 | 설명 |
|------|------|
| SPA | Single Page Application, 단일 페이지 애플리케이션 |
| JWT | JSON Web Token, 인증 토큰 |
| RBAC | Role-Based Access Control, 역할 기반 접근 제어 |
| ORM | Object-Relational Mapping, 객체-관계 매핑 |
| REST API | Representational State Transfer API |
| SSO | Single Sign-On, 통합 인증 |

---

## 5. ID 체계 용어

| 접두사 | 의미 |
|--------|------|
| REQ | Requirement, 요구사항 |
| REQ-FR | Functional Requirement, 기능 요구사항 |
| REQ-NFR | Non-Functional Requirement, 비기능 요구사항 |
| REQ-SEC | Security Requirement, 보안 요구사항 |
| DSN | Design, 설계 |
| TASK | Task, 작업 |
| TC | Test Case, 테스트 케이스 |

---

## 6. UI 용어

| 용어 | 설명 |
|------|------|
| 대시보드 | 전체 현황을 요약해서 보여주는 메인 화면 |
| 부서 탭 | 부서별로 카드를 분류해서 보여주는 탭 UI |
| 필터 | 특정 조건으로 카드를 걸러내는 기능 |
| 모달 | 화면 위에 떠서 나타나는 팝업 형태의 UI |
| 배지 | 숫자나 상태를 나타내는 작은 표시 |

---

## 7. 금지 용어

> ⚠️ 다음 용어는 시스템에서 **사용하지 않습니다**.

| 금지 용어 | 대체 용어 | 이유 |
|-----------|-----------|------|
| 관리 | 보드, 리스트 | 성과관리 느낌 배제 |
| 성과 | 정리됨 | 평가 느낌 배제 |
| 평가 | - | 평가 목적 아님 |
| 분석 | 훑어보기 | 가벼운 느낌 |
| 달성률 | - | 정량 지표 배제 |
| KPI | - | 성과 지표 배제 |
| 랭킹 | - | 경쟁 요소 배제 |

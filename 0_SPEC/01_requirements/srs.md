**문서 버전**: 2.0  
**작성일**: 2026-01-19  
**상태**: Approved

## 1. 개요

### 1.1 문서 목적
본 문서는 업무 개선 보드 시스템의 소프트웨어 요구사항을 정의한다.

### 1.2 프로젝트 개요
| 항목 | 내용 |
|------|------|
| 프로젝트명 | 업무 개선 보드 (Fix-it Board) |
| 서비스 컨셉 | 화이트보드 + 포스트잇을 온라인으로 옮긴 캐주얼 업무 개선 리스트 |
| 핵심 목적 | 성과관리가 아닌 **리스트 관리** |

### 1.3 서비스 포지셔닝

#### ❌ 하지 않는 것 (Anti-Requirements)
| ID | 항목 | 설명 |
|----|------|------|
| AR-01 | KPI/점수/평가 | 성과 측정 목적 기능 없음 |
| AR-02 | 부서 간 성과 랭킹 | 비교/경쟁 요소 배제 |
| AR-03 | 개인 평가 연동 | 인사 시스템 연계 없음 |
| AR-04 | 달성률/퍼센트 | 정량적 지표 표시 없음 |

#### ✅ 하는 것
- 현재 회사에 어떤 개선거리들이 있는지 파악
- 어느 부서에서 무엇을 고민 중인지 확인
- 아이디어/진행중/완료 상태 구분
- 방치된 항목 식별

## 2. 사용자 정의

### 2.1 사용자 유형

| ID | 사용자 | 역할 | 주요 활동 |
|----|--------|------|-----------|
| USER-01 | 일반 직원 | 등록자 | 개선거리 등록, 본인 항목 상태 변경 |
| USER-02 | 부서 담당자 | 관리자 | 부서 내 항목 관리, 상태 변경 |
| USER-03 | 경영자/임원 | 조회자 | 전체/부서별 항목 조회, 현황 파악 |
| USER-04 | 시스템 관리자 | 운영자 | 사용자/부서 관리, 시스템 설정 |

### 2.2 경영자 핵심 요구사항 (5대 질문)
| ID | 질문 | 대응 기능 |
|----|------|-----------|
| EQ-01 | 요즘 어디 부서에서 어떤 얘기들이 많이 올라오지? | 부서별 현황 요약 |
| EQ-02 | 이 중에서 아직 손도 못 댄 건 뭐야? | 떠올림 상태 필터 |
| EQ-03 | 오래 걸리는 건 왜 오래 걸릴까? | 오래된 항목 하이라이트 |
| EQ-04 | 이건 굳이 지금 안 해도 되는 거 아냐? | 멈춤 상태 리스트 |
| EQ-05 | 이미 끝난 건 어떤 식으로 정리됐지? | 정리됨 상태 조회 |

## 3. 기능 요구사항

### 3.1 개선 항목 등록 (REQ-FR-001)

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| REQ-FR-001-01 | 사용자는 개선 항목을 등록할 수 있다 | P1 |
| REQ-FR-001-02 | 제목은 필수 입력 항목이다 (최대 100자) | P1 |
| REQ-FR-001-03 | 부서는 로그인 사용자 기준 자동 선택된다 | P1 |
| REQ-FR-001-04 | 상태는 기본값 '💡 떠올림'으로 설정된다 | P1 |
| REQ-FR-001-05 | 메모(설명)는 선택 입력 항목이다 | P1 |
| REQ-FR-001-06 | 담당자는 선택 지정할 수 있다 | P2 |
| REQ-FR-001-07 | 관련 부서 태그를 복수 선택할 수 있다 | P2 |
| REQ-FR-001-08 | 파일(이미지/문서)을 첨부할 수 있다 | P2 |

#### 필수 입력 항목
| 필드 | 설명 | 입력 프롬프트 |
|------|------|---------------|
| 제목 | 한 줄 요약 | "뭐가 불편한가요?" |
| 메모 | 자유 서술 | "편하게 써주세요 (형식 없음)" |

#### 등록 UX 원칙
- **"나중에 다듬자"가 기본 철학**
- 최소한의 필수 입력만 요구
- 등록 버튼 문구: **[일단 올리기]**

### 3.2 상태 관리 (REQ-FR-002)

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| REQ-FR-002-01 | 항목은 5가지 상태 중 하나를 가진다 | P1 |
| REQ-FR-002-02 | 등록자는 본인 항목의 상태를 변경할 수 있다 | P1 |
| REQ-FR-002-03 | 부서 담당자는 부서 내 항목 상태를 변경할 수 있다 | P1 |
| REQ-FR-002-04 | 상태 변경 시 변경 이력이 자동 기록된다 | P1 |
| REQ-FR-002-05 | 상태 변경 시 메모를 남길 수 있다 | P2 |

#### 상태 정의 (캐주얼 버전)
| 상태 | 아이콘 | 의미 | 색상 |
|------|--------|------|------|
| IDEA | 💡 | 떠올림 (아이디어 단계) | 노랑 (#FCD34D) |
| REVIEWING | 👀 | 보고 있음 (검토 중) | 파랑 (#60A5FA) |
| IN_PROGRESS | 🛠️ | 만지는 중 (진행 중) | 초록 (#34D399) |
| ON_HOLD | ⏸️ | 잠깐 멈춤 (보류) | 회색 (#9CA3AF) |
| DONE | ✅ | 정리됨 (완료) | 민트 (#2DD4BF) |

# 추적성 매트릭스 (Traceability Matrix)
## 업무 개선 보드 시스템

---

## 1. 개요

추적성 매트릭스는 **요구사항 → 설계 → 작업 → 테스트** 간의 연결을 추적한다.

### 1.1 집계 기준 (P0-3)

> 📌 **본 문서의 요구사항 집계 기준**: **상위 REQ 단위** (REQ-FR-001)

| 기준 | 설명 | 예시 |
|------|------|------|
| 상위 REQ | 기능 그룹 단위 | REQ-FR-001 (항목 등록) |
| 하위 REQ | 세부 요구사항 단위 | REQ-FR-001-01 ~ 08 |

- 현재 문서는 **상위 REQ 기준**으로 커버리지를 집계
- 하위 REQ 세부 사항은 테스트 케이스(TC)에서 검증

---

## 2. 전체 추적성 체인

```
REQ-### → DSN-### → TASK-### → TC-###
   ↓          ↓          ↓         ↓
요구사항    설계      작업     테스트
```

## 7. 추적성 관리 프로세스

### 7.1 추적성 유지

1. **요구사항 추가/변경 시**
   - 해당 설계 문서 업데이트
   - 영향받는 작업 식별
   - 테스트 케이스 추가/수정

2. **설계 변경 시**
   - 영향받는 작업 재검토
   - 테스트 케이스 업데이트

3. **작업 완료 시**
   - 연결된 테스트 실행
   - 추적성 매트릭스 상태 업데이트

### 7.2 검증 시점

| 시점 | 검증 내용 |
|------|-----------|
| Sprint 시작 | 요구사항-작업 연결 확인 |
| PR 리뷰 | 작업-테스트 연결 확인 |
| 릴리스 전 | 전체 추적성 검증 |

# 인수 기준 (Acceptance Criteria)
## 업무 개선 보드 시스템

---

## 1. 개요

인수 기준은 각 기능이 **완료되었다고 판단하는 조건**을 정의한다.

---

## 2. 인증 기능

### AC-001: 로그인

```gherkin
Feature: 사용자 로그인

Scenario: 정상 로그인
  Given 유효한 이메일과 비밀번호를 가진 사용자가 있다
  When 로그인 폼에 자격증명을 입력하고 로그인 버튼을 클릭한다
  Then 대시보드 페이지로 이동한다
  And 사용자 이름과 부서가 헤더에 표시된다

Scenario: 잘못된 자격증명
  Given 잘못된 비밀번호로 로그인을 시도한다
  When 로그인 버튼을 클릭한다
  Then "이메일 또는 비밀번호가 올바르지 않습니다" 메시지가 표시된다

Scenario: 토큰 만료 시 자동 갱신
  Given 로그인된 사용자의 Access Token이 만료되었다
  When API 요청을 보낸다
  Then Refresh Token으로 새 Access Token이 자동 발급된다
  And 요청이 정상 처리된다
```

### AC-002: 로그아웃

```gherkin
Feature: 사용자 로그아웃

Scenario: 정상 로그아웃
  Given 로그인된 사용자가 있다
  When 로그아웃 버튼을 클릭한다
  Then 로그인 페이지로 이동한다
  And 저장된 토큰이 삭제된다
```

## 3. 항목 관리

### AC-003: 항목 등록

```gherkin
Feature: 개선 항목 등록

Scenario: 제목만 입력하여 등록
  Given 로그인된 직원이 있다
  When [일단 올리기] 버튼을 클릭한다
  And 제목을 입력한다
  And 등록 버튼을 클릭한다
  Then 새 항목이 💡 떠올림 상태로 등록된다
  And 내 부서에 자동 배정된다
  And 카드 리스트에 새 항목이 표시된다

Scenario: 제목 없이 등록 시도
  Given 등록 모달이 열려있다
  When 제목을 입력하지 않고 등록 버튼을 클릭한다
  Then "제목을 입력해주세요" 메시지가 표시된다
  And 항목이 등록되지 않는다
```

### AC-004: 항목 조회

```gherkin
Feature: 개선 항목 조회

Scenario: 전체 항목 조회
  Given 여러 부서의 항목이 존재한다
  When 전체 탭을 선택한다
  Then 모든 부서의 항목이 카드 형태로 표시된다
  And 최신순으로 정렬된다

Scenario: 부서별 필터링
  Given 생산팀, 품질팀의 항목이 존재한다
  When 생산팀 탭을 클릭한다
  Then 생산팀 항목만 표시된다
```

### AC-005: 상태 변경

```gherkin
Feature: 항목 상태 변경

Scenario: 본인 항목 상태 변경
  Given 내가 등록한 💡 떠올림 상태의 항목이 있다
  When 상세 모달에서 🛠️ 만지는 중 상태를 선택한다
  Then 상태가 🛠️ 만지는 중으로 변경된다
  And 상태 변경 이력이 기록된다
  And 카드의 상태 아이콘이 변경된다

Scenario: 타인 항목 상태 변경 불가 (일반 직원)
  Given 내가 일반 직원이고 타인이 등록한 항목이 있다
  When 해당 항목의 상태 변경을 시도한다
  Then "권한이 없습니다" 메시지가 표시된다
  And 상태가 변경되지 않는다
```

## 4. 대시보드

### AC-006: 대시보드 현황

```gherkin
Feature: 대시보드 현황 표시

Scenario: 요약 정보 표시
  Given 45건의 항목이 있고 5건이 정리됨, 5건이 멈춤 상태이다
  When 대시보드에 접속한다
  Then "전체 45건 / 정리됨 5건 / 멈춤 5건"이 표시된다

Scenario: 부서별 현황 표시
  Given 생산팀 15건, 품질팀 10건의 항목이 있다
  When 대시보드에 접속한다
  Then 부서별 바 차트에 건수가 표시된다

Scenario: 오래된 항목 표시
  Given 30일 이상 변경 없는 항목이 있다
  When 대시보드에 접속한다
  Then 오래된 항목 TOP 5가 표시된다
  And 각 항목의 경과 일수가 표시된다
```

## 5. 권한 관리

### AC-007: 역할별 권한

```gherkin
Feature: 역할별 권한 제어

Scenario: 일반 직원 권한
  Given 일반 직원으로 로그인했다
  Then 모든 항목을 조회할 수 있다
  And 본인 항목만 수정/삭제할 수 있다

Scenario: 부서 담당자 권한
  Given 생산팀 담당자로 로그인했다
  Then 모든 항목을 조회할 수 있다
  And 생산팀 항목을 수정/삭제할 수 있다
  And 타 부서 항목은 수정/삭제할 수 없다

Scenario: 경영자 권한
  Given 경영자로 로그인했다
  Then 모든 항목을 조회할 수 있다
  And 등록/수정/삭제 버튼이 표시되지 않는다
```

## 6. 반응형 UI

### AC-008: 모바일 대응

```gherkin
Feature: 반응형 UI

Scenario: 모바일 화면
  Given 모바일 기기로 접속한다 (width < 768px)
  Then 카드가 1열로 표시된다
  And 사이드바가 숨겨진다
  And 모든 기능이 정상 작동한다

Scenario: 태블릿 화면
  Given 태블릿으로 접속한다 (768px <= width < 1024px)
  Then 카드가 2열로 표시된다
```

## 7. 성능

### AC-009: 성능 기준

```gherkin
Feature: 성능 요구사항

Scenario: 페이지 로딩
  Given 정상적인 네트워크 환경이다
  When 대시보드 페이지에 접속한다
  Then 3초 이내에 페이지가 로딩된다

Scenario: API 응답
  Given 정상적인 서버 상태이다
  When 항목 목록 API를 호출한다
  Then 1초 이내에 응답을 받는다
```
